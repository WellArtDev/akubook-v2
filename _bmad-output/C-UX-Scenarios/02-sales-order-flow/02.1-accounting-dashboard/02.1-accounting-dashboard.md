## Interactions

### On Load
1. Fetch auto-posting stats for today/week/month
2. Fetch pending review entries (limit 5)
3. Fetch quick stats (AR, AP, Cash)
4. Display success indicator if 95%+ achieved

### On Period Change
1. Update all widgets for selected period
2. Refresh auto-posting stats
3. Refresh review queue

### On "Review →" Click (Entry Card)
1. Navigate to Journal Entry Detail page (02.3)
2. Pass entry ID as parameter
3. Open in same tab

### On "Lihat Semua →" Click
1. Navigate to Journal Entry List page (02.2)
2. Filter: Status = "Pending Review"
3. Open in same tab

### On Entry ID Click (e.g., "JE#12345")
1. Navigate to Journal Entry Detail page (02.3)
2. Pass entry ID as parameter

### On Source Chain Click (e.g., "SO#12345")
1. Open source document in modal or new tab
2. Show full transaction chain

### On Quick Action Click
1. Navigate to respective module/page
2. Maintain context (return to dashboard)

## Data Requirements

### Auto-Posting Stats API
```
GET /api/accounting/auto-posting-stats?period={today|week|month}

Response:
{
  "period": "today",
  "total_transactions": 12,
  "auto_posted": 12,
  "manual_posted": 0,
  "failed": 0,
  "percentage": 100,
  "target_met": true
}
```

### Review Queue API
```
GET /api/accounting/journal-entries?status=pending_review&limit=5

Response:
{
  "entries": [
    {
      "id": "JE#12345",
      "source_chain": ["SO#12345", "INV#12345"],
      "amount": 15500000,
      "posted_at": "2026-05-12T16:45:00Z",
      "summary": {
        "debit": [{"account": "AR", "amount": 17050000}],

          {"account": "Revenue", "amount": 15500000},

```

```
GET /api/accounting/quick-stats

Response:
{
  "ar_balance": 125500000,

  "cash_balance": 45800000

```

## Validation Rules

**Period Selector:**
- Must be valid fiscal period
- Cannot select future periods
- Default: Current month

**Review Queue:**
- Show max 5 entries
- Sort by posted_at DESC (newest first)
- Only show "Pending Review" status

## Edge Cases

**No Pending Reviews:**
- Show: "✓ Semua transaksi sudah direview"
- Hide review queue widget or show empty state

**Below 95% Target:**
- Change status card border to yellow
- Show warning icon instead of checkmark
- Text: "⚠️ Target belum tercapai - perlu investigasi"

**Failed Auto-Posting:**
- Show separate "Failed" count
- Red indicator
- Link to failed entries list

**First Day of Month:**
- "This Month" stats show current day only
- Note: "Baru hari pertama bulan ini"

## Notes

**Design Decisions:**
- Hero section focuses on THE metric (95%+ auto-posting)
- Review queue shows just enough detail to decide priority
- Quick stats provide context without overwhelming
- Clear visual hierarchy: Status → Review → Actions

**Why This Works:**
- Sari sees success immediately (100% today)
- Monthly target (95%) prominently displayed
- Review queue is actionable (not just informational)
- Supports her workflow: Check → Review → Approve

# Page 12.3: e-Faktur Generation

**Scenario:** 12 - e-Faktur & Tax Reporting  
**Page:** 12.3 - e-Faktur Generation  
**Created:** 2026-05-13  
**Status:** ✅ Specified

---

## Page Purpose

Auto-generate e-Faktur for selected invoices with validation

---

## Key Components

### 1. Generation Progress
- Progress bar: "Generating e-Faktur... 25/52 complete"
- Current invoice being processed
- Estimated time remaining

### 2. Generation Summary
- Total selected: 52 invoices
- Successfully generated: 48
- Failed: 4 (with reasons)
- e-Faktur numbers assigned: Range display

### 3. Generation Results Table
Columns: Invoice# | Customer | e-Faktur# | DPP | PPN | Status

**Success Row:**
- Green checkmark icon
- e-Faktur number assigned
- Status: "Generated ✓"

**Failed Row:**
- Red X icon
- Error reason: "NPWP invalid" / "e-Faktur range exhausted" / "DJP connection failed"
- Action: "Retry" or "Fix & Retry"

### 4. e-Faktur Number Assignment
- Format: 010.000-26.00012345
- Range: From DJP allocation
- Sequential numbering
- Display: "Using range: 00012345 - 00012396"

### 5. Actions
- Review e-Faktur → Navigate to e-Faktur Review (12.4)
- Retry Failed → Re-attempt failed generations
- Cancel → Return to invoice list

---

## Generation Logic

**For each invoice:**
1. Validate NPWP (15 digits, valid format)
2. Calculate DPP (invoice amount excluding PPN)
3. Determine PPN rate (11% or 12% based on invoice date)
4. Calculate PPN amount
5. Assign e-Faktur number (sequential from range)
6. Generate e-Faktur record
7. Update invoice status

**Validation Rules:**
- NPWP must be valid
- e-Faktur range must have available numbers
- Invoice must not already have e-Faktur
- Customer must be PKP (Pengusaha Kena Pajak)

---

## Interactions

- Auto-start generation on page load
- Real-time progress updates
- Show errors immediately
- "Review e-Faktur" enabled when complete
- "Retry Failed" for failed items

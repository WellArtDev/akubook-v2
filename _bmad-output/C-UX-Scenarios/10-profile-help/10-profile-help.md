# 10: User's Profile & Help

**Project:** AkuBook  
**Created:** 2026-05-12  
**Method:** Whiteport Design Studio (WDS)

---

## Transaction (Q1)

**What this scenario covers:**
Update profile settings → access contextual help documentation when stuck

---

## Business Goal (Q2)

**Goal:** Supporting (All Goals) — Reduce Support Burden  
**Objective:** Users self-service profile management and help, reducing support calls

---

## User & Situation (Q3)

**Persona:** Any user (Sari, Budi, Pak Hendra, Staff)  
**Situation:** Need to update profile (password, email, preferences) or stuck on a feature and need help documentation.

---

## Driving Forces (Q4)

**Hope:** Self-service profile management, contextual help available when stuck, issue resolved without calling support.

**Worry:** Can't find help, need to call support, productivity blocked, learning curve too steep.

---

## Device & Starting Point (Q5 + Q6)

**Device:** Desktop, Mobile  
**Entry:** As-needed — profile update or when stuck on a feature (contextual help link from any page).

---

## Best Outcome (Q7)

**User Success:**
Profile updated successfully, help documentation found and issue resolved. No support call needed, productivity maintained.

**Business Success:**
Reduced support burden (users self-service), user autonomy increased, learning curve mitigated. Support team focuses on complex issues, not basic questions.

---

## Shortest Path (Q8)

1. **Profile Settings** — User updates profile (password, email, preferences), saves changes
2. **Help/Documentation** — User clicks "Help" from any page, searches for feature, finds contextual help, resolves issue ✓
